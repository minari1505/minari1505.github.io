## Claude가 기대와 다르게 답할 때

Claude를 쓰다 보면 답변이 기대와 다르게 나올 때가 있습니다. 너무 일반적이거나, 길이가 맞지 않거나, 형식을 지키지 않거나, 그럴듯하지만 틀린 정보를 줄 수도 있습니다.

이것은 실패라기보다 **요청 방식을 조정할 기회**에 가깝습니다. Claude와 잘 협업하려면 첫 prompt 하나로 완벽한 결과를 얻으려 하기보다, 응답을 보고 방향을 조정하는 습관이 필요합니다.

## 흔한 문제와 해결 방법

| 문제 | 왜 발생하나 | 시도해볼 것 |
|---|---|---|
| 답변이 너무 일반적임 | 내 상황에 대한 맥락이 부족함 | 대상, 역할, 제약 조건을 추가하기 |
| 답변이 너무 길거나 짧음 | Claude가 적절한 길이를 추측함 | "두 문단", "100단어 이하", "길이 제한 없음"처럼 명시하기 |
| 원하는 형식을 따르지 않음 | 무엇을 원하는지는 알지만 어떻게 보여줄지 모름 | 예시를 제공하거나 구조를 구체적으로 설명하기 |
| 자신감 있어 보이지만 틀린 정보가 나옴 | 특정 사실이나 niche topic에서 plausible but incorrect 정보가 생성될 수 있음 | 중요한 정보는 독립 검증하고, source/citation 또는 confidence level 요청하기 |
| 톤이 맞지 않음 | Claude는 기본적으로 helpful하고 professional한 톤으로 답함 | "더 대화체로", "권위 있고 공식적으로"처럼 톤을 평이한 말로 설명하기 |

예를 들어 다음 prompt는 너무 일반적입니다.

```text
프로젝트 지연에 대한 이메일을 써줘.
```

더 좋은 prompt는 상황과 독자를 알려줍니다.

```text
엔터프라이즈 고객에게 소프트웨어 통합이 2주 지연된다는 이메일을 써줘.
그들은 지금까지 기다려줬지만 이번이 두 번째 지연이야.
전문적이되 사과하는 톤으로 작성해줘.
```

Claude가 더 잘 답하려면 추측해야 하는 부분을 줄여야 합니다.

## Iteration mindset

Claude와 일할 때 중요한 전환은 첫 prompt를 "완성본 요청"이 아니라 **대화의 시작**으로 보는 것입니다.

효과적으로 Claude를 쓰는 사람들은 보통 이렇게 작업합니다.

1. Claude에게 초안을 요청합니다.
2. 나온 결과에서 좋은 점과 부족한 점을 확인합니다.
3. 구체적인 피드백을 줍니다.
4. 필요한 경우 예시나 추가 맥락을 제공합니다.
5. 대화가 너무 엇나가면 새 채팅에서 더 명확한 prompt로 다시 시작합니다.

피드백은 구체적일수록 좋습니다.

```text
짧게 만들어줘.
```

보다:

```text
첫 두 문단은 빼고, 결론을 더 행동 지향적으로 바꿔줘.
```

가 더 좋습니다.

## AI Fluency란?

**AI Fluency**는 AI 도구를 효과적으로 협업에 활용하는 능력입니다. 단순히 어떤 버튼을 누르는지 아는 것이 아니라, 다양한 상황에서 AI를 어떻게 맡기고, 설명하고, 평가하고, 책임 있게 사용할지 판단하는 능력입니다.

강의에서는 AI Fluency의 4D Framework를 소개합니다. 이 framework는 Rick Dakan 교수와 Joseph Feller 교수의 연구 협업에서 나온 것으로, AI와 효과적으로 협업하기 위한 네 가지 역량을 제시합니다.

| 역량 | 의미 |
|---|---|
| **Delegation** | 무엇을 사람이 하고, 무엇을 AI가 하며, 어떻게 역할을 나눌지 결정하는 능력 |
| **Description** | AI에게 원하는 결과, 과정, 행동 방식을 명확히 설명하는 능력 |
| **Discernment** | AI의 출력, 과정, 행동을 비판적으로 평가하는 능력 |
| **Diligence** | AI를 책임 있고 윤리적으로 사용하는 능력 |

이전 레슨의 prompt framework도 4D와 연결됩니다.

- 역할과 목표를 알려주는 것: Description
- 결과를 검토하고 고치는 것: Discernment
- 고위험 작업에서 사실을 검증하는 것: Diligence
- 어떤 일을 Claude에게 맡길지 정하는 것: Delegation

Claude를 잘 쓰는 것은 prompt 기술만의 문제가 아닙니다. **무엇을 맡기고, 어떻게 설명하고, 어떻게 검토하고, 어떤 책임을 질지**를 함께 판단하는 일입니다.

## 내 업무에 Claude가 잘 맞는지 평가하기

Claude를 업무에 더 많이 쓰기 시작하면 이런 질문이 생깁니다.

> Claude가 이 특정 작업을 정말 잘할까?

이때 필요한 것이 **eval**입니다. Evals는 Claude가 내 업무에서 중요한 특정 작업을 얼마나 잘 수행하는지 체계적으로 확인하는 방법입니다.

거창한 인프라가 없어도 간단히 시작할 수 있습니다.

## 간단한 eval 방법

### 1. 예시 모으기

내가 반복적으로 하는 작업의 예시를 5-10개 모읍니다.

- 내가 직접 작성한 이메일
- 기존 보고서
- 내가 만든 분석 자료
- 자주 쓰는 문서 양식

### 2. 테스트 prompt 만들기

그 작업을 Claude가 수행하도록 prompt를 작성합니다. 실제 업무에서 Claude에게 제공할 만한 맥락을 넣습니다.

### 3. 결과 비교하기

Claude의 결과를 내가 만든 예시와 비교합니다.

확인할 질문:

- 핵심 정보를 잘 담았는가?
- 톤과 스타일이 적절한가?
- 빠진 내용은 무엇인가?
- 사람이 반드시 검토해야 할 부분은 어디인가?
- 예시를 더 주면 좋아질 것인가?

### 4. 접근 방식 개선하기

비교 결과를 바탕으로 prompt를 조정합니다.

- 더 많은 맥락 추가
- 좋은 결과 예시 제공
- 원하는 형식 명확히 지정
- 사람이 검토해야 하는 단계 분리

## 예시: 데이터 분석 eval

데이터 분석 작업에 Claude를 쓰고 싶다면 다음처럼 평가할 수 있습니다.

1. 내가 과거에 직접 분석한 dataset을 찾습니다.
2. Claude에게 같은 분석을 요청하는 prompt를 작성합니다.
3. Claude의 결과와 내 기존 분석을 비교합니다.
4. Claude가 숫자는 맞추지만 전체 pattern을 놓치는지, 혹은 trend는 잘 잡지만 세부 계산이 약한지 확인합니다.
5. 그 pattern에 맞춰 prompt를 개선합니다.

이런 가벼운 eval은 Claude를 어디에 믿고 쓸 수 있는지, 어디에서 review와 refinement에 집중해야 하는지 감각을 길러줍니다.

## 핵심 정리

Claude와 잘 협업하려면 첫 답변을 그대로 받아들이기보다 반복적으로 조정해야 합니다.

- 답변이 일반적이면 맥락을 추가합니다.
- 길이와 형식은 명확히 지정합니다.
- 중요한 사실은 독립적으로 검증합니다.
- 톤은 구체적인 말이나 예시로 설명합니다.
- 첫 prompt는 대화의 시작점입니다.
- AI Fluency는 Delegation, Description, Discernment, Diligence를 함께 기르는 능력입니다.
- 간단한 eval을 통해 Claude가 내 업무에 얼마나 잘 맞는지 확인할 수 있습니다.
