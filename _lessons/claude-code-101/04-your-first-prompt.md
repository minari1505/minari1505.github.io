## 학습 목표

- auto-accept와 approval 모드의 차이와 전환 방법 이해하기
- Plan Mode가 언제 유용한지 파악하기
- 구체적인 prompt로 복잡한 작업을 안전하게 시작하는 방법 익히기

## Auto-Accept vs. Approval

Claude가 제안하는 모든 파일 변경을 자동으로 수락할지, 매번 명시적 허가를 요청하게 할지 선택할 수 있습니다. **Shift + Tab**으로 모드를 순환합니다.

- **Approval 모드**: 파일 수정이나 명령 실행 때마다 Claude가 허가를 요청합니다.
- **Auto-accept 모드**: 파일 수정은 자동 승인되지만, 명령은 여전히 허가가 필요합니다.

정답은 없습니다 — 자신이 편한 방식으로 고르면 됩니다.

## Plan Mode

Shift + Tab 메뉴 안에 **Plan Mode**가 있습니다. Plan Mode는 prompt를 받아 읽기 전용 도구로 코드베이스를 분석하고 제안한 구현을 조사합니다. 중간에 확인 질문을 던지고, 실행 가능한 상세 계획을 돌려줍니다.

Plan Mode는 복잡한 변경을 계획하거나 안전하게 코드 리뷰를 할 때 특히 좋습니다. 여러 단계로 이루어진 기능 구현을 요청하는 경우가 많은데, 바로 그런 상황에서 Plan Mode가 빛을 발합니다.

## 예시: 다크 모드 토글 추가

예를 들어 다크 모드 토글이 필요한 애플리케이션이 있다고 합시다. 프로젝트 루트에서 `claude`를 실행하고 Shift + Tab을 몇 번 눌러 Plan Mode로 진입한 뒤 이렇게 씁니다.

```text
내 앱 전체에 다크 모드를 구현해야 해. 헤더에 라이트/다크 모드를 전환하는 토글 스위치를 만들어줘.
기존 라이트 테마를 기준으로 대비가 좋은 색을 찾아줬으면 해.
```

Claude가 계획을 세우게 두세요. 계획을 검토해 괜찮으면 수락하고, 단계마다 승인을 받으며 진행하게 합니다. 끝나면 Claude가 무엇을 어떻게 했는지 정확히 확인할 수 있습니다.

## 핵심 정리

- Claude Code를 쓸 때는 prompt를 최대한 구체적으로 작성합니다.
- 모든 단계에 관여하고 싶으면 Approval 모드를, 편하게 맡기고 싶으면 Auto-accept를 선택합니다 (Shift + Tab으로 전환).
- 코드 작성 전에 Claude가 원하는 바를 깊게 파고들게 하려면 Plan Mode를 활용합니다.
