## 학습 목표

- Next Token Prediction이 생성형 AI의 핵심 메커니즘이며, 왜 **유창함과 환각을 동시에** 만드는지 설명하기
- 작업을 Next Token Prediction 연속선(익숙한 길 vs 새로운 영역) 위에 놓기
- **구체성(이름·날짜·인용·통계)**이 조작(fabrication)이 집중되는 구간임을 식별하기
- 이 한계를 완화하는 제품 기능(citation, 불확실성 신호, 제약 생성, generator-verifier 패턴) 인식하기

## 대규모 자동완성

생성형 AI는 검색 엔진보다 **엄청나게 정교한 자동완성(autocomplete)**에 가깝습니다. 무엇 뒤에 무엇이 오는 경향이 있는지를 바탕으로 답을 단어 하나씩 써 내려갑니다. **바로 이 하나의 속성이 유창함과 환각을 동시에** 줍니다.

### 이것이 가능하게 하는 것 (능력)

- 거의 모든 스타일·형식으로 유창하고 자연스러운 텍스트
- 서로 먼 분야의 아이디어를 빠르게 종합
- 모델이 이전에 본 것과 닮은 작업에서 강한 성능
- 어떤 흐름(이야기, 논증, 코드 블록)이든 일관되게 이어가기

### 특징적으로 실패하는 곳 (한계)

- **환각(Hallucination)**: 그럴듯한 이어짐이 항상 참인 것은 아님
- **작화(Confabulation)**: 빈틈을 표시하지 않고 그럴듯한 내용으로 메움
- **비일관성(Inconsistency)**: 샘플링 때문에 같은 prompt가 다른 출력을 낼 수 있음
- **잘못된 확신(Misplaced confidence)**: 매끄러운 문장이 추측을 감쌀 수 있음

### 한계를 밀어내는 제품 기능

- **Citation·source grounding**: 무엇이 근거 있는지 vs 생성된 것인지 추적
- **불확실성 신호**: 모델이 자신의 불안정함을 스스로 표시
- **제약 생성(constrained generation)·skills**: 조작이 스며들 공간을 좁힘

## 조작은 "구체성"에 집중된다

핵심 통찰: **조작은 구체성에서 집중적으로 발생**합니다 — 이름, 날짜, 통계, 인용, URL, 인용문. 주장이 정밀할수록 검증이 더 필요합니다.

- **능력 구간**: 모델이 여러 번 본 패턴을 닮은 작업 (요약, 재구성, 흔한 개념 설명)
- **한계 구간**: 새롭거나 희소한 영역, 그리고 "참"과 "그럴듯함"을 구분해야 하는 모든 곳

## 실습 아이디어: 검증 테스트

내가 가장 전문성 있는 주제를 골라, 검증 가능한 구체적 사실 5개(직함, 출판일, 통계, 사양, 직접 인용, URL)를 준비한 뒤 세 가지를 시도해봅니다.

1. **능력 구간** — 잘 알려진 개념을 설명·요약시키기. 유창하고 대체로 정확함을 느껴보기.
2. **구체성의 압박** — 출처 3개, 저자명, 정확한 수치, URL을 요구하고 하나하나 검증. 5점 만점 채점.
3. **샘플링 확인** — 같은 요청을 새 대화에서 다시 실행해 두 출력을 비교. 무엇이 유지되고 무엇이 바뀌는지 관찰.

## 핵심 정리

- Next Token Prediction은 생성형 AI가 "무엇 뒤에 무엇이 오는지"를 바탕으로 단어 하나씩 답을 쓴다는 사실입니다.
- 능력 구간은 익숙한 패턴(요약·재구성·흔한 개념), 한계 구간은 새롭거나 희소한 영역과 "참 vs 그럴듯함" 구분이 필요한 곳입니다.
- **조작은 구체성(이름·날짜·통계·인용·URL)에 집중**됩니다 — 정밀한 주장일수록 검증하세요.
- citation·불확실성 신호·제약 생성·generator-verifier 루프는 이 한계를 밀어내는 장치입니다.
- **4D 연결**: Next Token Prediction은 **Discernment(분별)**의 토대입니다 — 출력이 "생성된" 것임을 알면 어떤 종류의 검증을 적용할지 알 수 있습니다.
