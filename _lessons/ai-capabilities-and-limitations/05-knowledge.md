## 학습 목표

- AI 모델의 지식이 학습 중 어떻게 형성되고 왜 고정된 컷오프를 갖는지 설명하기
- 어떤 주제가 능력 구간(자주·최근·일관)인지, 가장자리(드묾·컷오프 이후·niche·논쟁적)인지 예측하기
- staleness, 고르지 않은 coverage, 상속된 bias, source amnesia를 특징적 지식 실패로 식별하기
- 이 한계를 다루는 제품 기능(web search, retrieval/RAG, tool use) 인식하기

## 모델이 읽은 것, 그리고 읽기를 멈춘 시점

모델은 **학습 중 노출된 것만** 압니다. 기본적으로 실시간 브라우징도, 체험된 경험도 없고, **knowledge cutoff에서 딱 멈춥니다.** 그래서 실용적인 질문은 "AI가 이걸 아는가?"가 아니라 **"이것이 모델이 읽은 것 안에서 얼마나 잘 대표되어 있었나?"**입니다.

### 이것이 가능하게 하는 것 (능력)

- 대단히 넓은 일반 지식
- 잘 대표된 분야에서의 깊은 역량
- 분야를 가로지르는 연결 (embedding: 관련 개념이 가까이 모임)

### 특징적으로 실패하는 곳 (한계)

- **Knowledge cutoff**: 학습 종료 이후의 것은 없음
- **Staleness(진부화)**: 학습 시점엔 참이었어도 지금은 아닐 수 있음
- **고르지 않은 coverage**: niche 분야·지역 지식이 취약
- **상속된 bias**: 기본값이 학습 데이터의 사각지대를 반영
- **Source amnesia**: "어디선가 읽었다"는 citation이 아님

### 한계를 밀어내는 제품 기능

- **Web search**: 시간에 민감한 질문에서 컷오프를 우회
- **Retrieval(RAG)/MCP**: 학습된 적 없는 자료를 끌어옴
- **Tool use**: 실제 계산기·데이터베이스·API 호출
- **명시적 cutoff 고지**: 무엇을 재확인할지 알 수 있음

## 능력 구간 vs 한계 구간

- **능력 구간**: 학습 데이터에 **자주·최근·일관되게** 나타난 주제 (주류 주제, 인기 언어)
- **한계 구간**: 드물거나, 컷오프 이후이거나, niche·지역적·논쟁적인 주제

## 실습 아이디어: 아웃사이더 테스트

내 분야에서 (1) 주류·문서화·안정적인 주제 2개, (2) niche·지역·최근·급변하는 주제 2개, (3) 외부인이 자주 틀리는 "기본 가정" 1개를 적고 세 가지를 시도합니다.

- **Probe 1: Coverage** — 주류 vs niche 주제를 물어 깊이·정확도, 그리고 불확실성 신호 방식을 비교.
- **Probe 2: Staleness** — 최근 바뀐 것을 물어 컷오프를 인정하는지, 진부한 정보를 최신처럼 제시하는지 관찰.
- **Probe 3: 기본 가정** — 내 가정을 직접 말하지 않고, AI가 외부인의 관점을 기본값으로 삼는지 드러나는 질문.

## 핵심 정리

- AI의 지식은 전적으로 학습 데이터에서 오고 **컷오프에서 고정**됩니다. 도구 없이는 그 이후 정보에 접근할 수 없습니다.
- 능력 구간은 자주·최근·일관되게 나타난 주제, 한계 구간은 드물거나 컷오프 이후·niche·지역·논쟁적 주제입니다.
- 특징적 실패: staleness, 고르지 않은 coverage, 상속된 bias, source amnesia.
- web search·RAG/MCP·tool use는 학습되지 않은 정보에 접근시켜 이 간극을 메웁니다.
- **4D 연결**: 지식의 불균형은 **Delegation**의 핵심입니다 — 모델이 어디서 든든하고 어디서 얄팍한지 알면, 언제 맡기고 언제 맥락을 직접 주고 언제 다른 곳으로 갈지 판단할 수 있습니다.
