## 학습 목표

- context window를 **고정 크기 그릇**으로 이해하고, 긴 문서·긴 대화·세션 간 기억에 무엇을 의미하는지 설명하기
- 다른 속성의 점진적 저하와 달리 이 속성이 갖는 **"절벽(cliff)"** 성격 인식하기
- "맥락을 지렛대로" 쓰는 전략(중요 자료 앞배치, 긴 작업 쪼개기, 핵심 맥락 재공급) 적용하기
- 이 한계를 다루는 제품 기능(memory, compaction, projects/workspaces, 큰 window) 인식하기

## context window: AI의 작업 기억

AI가 주의를 두는 모든 것은 **context window**라는 고정 크기 작업 공간 안에 있습니다. 그 안의 것은 참조할 수 있고, 밖의 것은 참조할 수 없습니다. 이 제약은 다른 속성들과 달리 **딱딱한 경계**를 갖습니다 — **되다가 갑자기 안 됩니다.**

### 이것이 가능하게 하는 것 (능력)

- 세션 안에서 즉각적 적응 (스타일 가이드·샘플이 바로 반영됨)
- 내 자료로 작업 (내 문서, 내 제약 조건)
- 구체성을 통한 정밀함 — **맥락이 곧 지렛대(context is leverage)**

### 특징적으로 실패하는 곳 (한계)

- **딱딱한 길이 한계** — 초과 시 조용히 잘림(silent truncation)
- **중간에서 길을 잃음(lost in the middle)** — 주의는 window 전체에 균일하지 않음
- **기본적으로 지속 기억 없음** — 각 세션은 0에서 시작
- **사용자로부터 학습하지 않음** — 교정은 모델이 아니라 context만 바꿈

### 한계를 밀어내는 제품 기능

- **Memory**: 세션을 넘어 사실을 유지
- **Compaction/요약**: 오래된 턴을 압축해 공간 확보
- **Projects/workspaces**: 상시 문서를 안정적으로 context에 유지
- **Skills**: 필요할 때까지 context 사용 최소화
- **더 큰 context window**: 절벽을 더 멀리 밀어냄

## "절벽" 성격

이 속성의 실패는 점진적 저하가 아니라 **절벽**입니다. **조용한 잘림(silent truncation)**이 대표적 실패 방식이며, 항상 경고받지는 못합니다. 또한 모델은 내 교정으로부터 **학습하지 않고**, 오직 현재 context에 있는 것에만 반응합니다.

- **능력 구간**: 자료가 여유 있게 들어가고, 세션이 최신이며, 관련 맥락을 직접 제공할 때
- **한계 구간**: 아주 긴 문서·대화, 세션 간 연속성 기대, 핵심 정보를 긴 입력의 중간에 묻을 때

## 실습 아이디어: Before-and-After

- **Probe 1: 콜드 스타트 vs 맥락** — 맥락 없이 요청 → 저장. 새 대화에서 스타일 가이드·예시·제약을 앞배치해 다시 요청 → 비교.
- **Probe 2: 중간에서 길 잃기** — 긴 문서 중간에 중요한 지시 하나를 묻고, 그에 의존하는 질문. 놓치는지 확인 후, 지시를 맨 위로 옮겨 다시 질문.
- **Probe 3: 백지 상태** — 뭔가를 가르치거나 교정한 뒤, 새 대화에서 그걸 기억한다고 가정한 질문 → 0에서 시작함을 관찰.

## 핵심 정리

- Working Memory는 AI가 참조할 수 있는 **고정 크기 context window**를 갖는다는 사실입니다.
- 능력 구간은 자료가 여유 있고 맥락을 직접 제공할 때, 한계 구간은 아주 길거나 세션 간 연속성을 기대하거나 핵심을 중간에 묻을 때입니다.
- 이 속성은 점진적 저하가 아니라 **절벽**이며, 조용한 잘림이 주요 실패 방식입니다.
- 모델은 교정으로부터 학습하지 않고 현재 context에만 반응합니다.
- **4D 연결**: Working Memory는 **Description**이 작용하는 대상입니다 — window 작동 방식을 알면 언제 맥락을 앞배치하고 언제 새로 시작할지 알 수 있습니다.
