## 학습 목표

- Word, Excel, PowerPoint, Outlook 안에서 Claude가 하는 일 알기
- 하나의 Claude 대화로 앱 간에 작업 넘기기
- Cowork에서 할지, 문서 안에서 할지 판단하기

## 핵심

- **Claude가 문서 안에 산다.** Word, Excel, PowerPoint, Outlook에 add-in으로 나타나, 열려 있는 파일과 공간에서 작업합니다.
- **하나의 대화가 앱 간 맥락을 나른다.** Excel에서 분석을 만들어 PowerPoint에 넘겨 슬라이드로. Word memo를 소스 삼아 Outlook 답장 초안으로.
- **Cowork의 Claude와 M365의 Claude는 다른 순간을 위한 다른 도구.** Cowork는 여러 소스에서 완성 결과물을 만들고, M365는 이미 작업 중인 파일을 다듬고·디버깅하고·형태를 잡습니다.

## 앱별로 하는 일

- **Excel.** 데이터 분석, 수식 작성, `#REF!`·순환 참조 디버깅, 모델 안 깨고 시나리오 테스트, 템플릿에서 시트 생성, 특정 셀 인용. 강력한 한 수: "Q3 시트의 실적을 같은 워크북의 Q3 계획과 비교해, F열에 항목별 분산 코멘트를 써줘."
- **PowerPoint.** 슬라이드 마스터·폰트·색 구성을 읽고 맞춰 슬라이드 생성. 네이티브·편집 가능 차트(붙여넣은 이미지 아님). 선택한 슬라이드에서 작동. 강력한 한 수: "방금 Excel에서 한 분석을 우리 덱 템플릿으로 QBR용 3슬라이드 덱으로 만들어줘."
- **Word.** 제자리에서 초안·수정·재포맷. 코멘트·변경 내용 추적과 작업. 연결 소스에서 맥락을 끌어와 초안을 근거화.
- **Outlook.** 나머지 작업 맥락으로 수신 메일 분류. 이전 스레드·캘린더·최근 결정을 반영한 답장 초안.

## 하나의 대화로 앱 간 이동

- **Outlook → Word.** 받은 브리프를 "우리 회사 템플릿으로 memo 시작해줘" — Word가 첨부와 이메일 스레드를 사이드바에 로드한 채 실행.
- **Word → Excel.** "옵션 2의 시장 규모 모델을 만들어줘" — Excel이 Word에서 읽은 브리프의 가정을 끌어와 다중 탭 모델 생성.
- **Excel → PowerPoint.** "이걸 고객 템플릿의 운영위 덱으로 만들어줘" — 슬라이드 마스터에 네이티브 차트로 빌드.
- **다시 Outlook.** "목요일 전 팀과 30분 잡아줘" — 참석자 포함 초대 초안이 전송 대기.

## M365 vs Cowork, 언제 무엇을

- **Cowork** — 여러 소스에서 끌어와 결과물로 끝나는 작업. 20개 소스 파일로 브리프 만들기, Salesforce+Slack 3채널로 보고서 조립, 일정 워크플로.
- **M365 안 Claude** — Office 파일 자체에서 제자리 편집하고 앱 간 맥락을 나를 때.

대부분의 실제 작업은 둘 다 씁니다. Claude가 덱 초안을 만들면 PowerPoint에서 다듬고, 데이터를 모델로 끌어오면 Excel에서 이상치를 조사하게 합니다.

## 핵심 정리

- Claude는 Word·Excel·PowerPoint·Outlook 안에 add-in으로 들어가 열린 파일에서 작동합니다.
- 하나의 대화가 앱 간 맥락을 날라, Excel 분석 → PowerPoint 슬라이드 같은 이동이 매끄럽습니다.
- 여러 소스·결과물이면 Cowork, 파일 자체를 다듬으면 M365 — 대부분은 둘을 함께 씁니다.
