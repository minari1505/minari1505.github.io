## 학습 목표

- Claude Desktop app에서 Cowork를 열고 작업 폴더를 지정하기
- 작업이 있는 앱을 커넥터로 연결하기
- Claude가 무엇을 하기 전에 묻고 무엇은 묻지 않는지 이해해 안심하고 위임하기

## Cowork 설치

Cowork는 Mac·Windows의 **Claude Desktop app** 안에서 동작합니다. 없다면 `claude.com/download`에서 설치하고, 앱을 열어 로그인한 뒤 우측 상단 모드 선택기에서 Cowork를 찾습니다. 보이지 않으면 유료 plan이나 최신 버전이 필요할 수 있습니다.

## 폴더 지정하기

프롬프트 바에서 **Work in a project**를 클릭하고 컴퓨터의 폴더를 고릅니다. 이것이 새 작업마다 하는 **가장 중요한 설정 선택**입니다. 폴더가 곧 작업이 사는 곳이기 때문입니다. Claude는 폴더 안의 모든 파일(Word, Excel, PDF, PowerPoint 등)을 읽고, 완성된 출력을 같은 위치에 저장합니다.

- **하나의 프로젝트로 좁혀진 폴더를 고르세요.** 전체 문서 폴더가 아니라 그 작업에 필요한 파일이 든 폴더면 됩니다.
- 폴더는 Claude가 **읽기 AND 쓰기** 권한을 갖는 곳입니다. 파일을 열고, 수정하고, 새로 만들고, 정리할 수 있습니다. Chat과의 핵심 차이입니다 — Chat은 업로드한 것을 읽을 뿐 컴퓨터에 저장하지 못합니다.
- **맥락이 있는 실제 폴더**를 고르세요. 원본 자료, 관련 문서, 템플릿이 있을 때 가장 잘 작동합니다.
- **클라우드 파일은 다르게 동작**합니다. 커넥터마다 가능한 범위가 다릅니다. 기본 Google Drive·M365 커넥터는 대개 읽기·검색만 가능합니다. 활성화 시 각 커넥터 설명을 확인하세요. 문서를 만들고 반복 작업하기엔 로컬 작업 폴더가 여전히 가장 안정적입니다.

## 커넥터 추가

커넥터는 Claude가 이미 작업이 일어나는 앱(이메일, 캘린더, 팀 메신저, CRM, 클라우드 저장소)에 닿는 방법입니다. **Customize** 영역에서 한 번 설정하고, 작업마다 필요한 것을 켜고 끕니다.

대부분 먼저 설정하는 커넥터:

- **이메일·캘린더** (M365 Outlook 또는 Gmail) — 회의 맥락 추출, 후속 이메일 작성, 과거 스레드 찾기
- **메신저** (Slack 또는 M365 Teams) — 채널 히스토리 검색, 팀 논의 종합
- **클라우드 저장소** (SharePoint/OneDrive via M365, Google Drive, Box)
- **CRM·프로젝트 도구** — Notion, Salesforce, HubSpot, Asana, Linear 등

켜면 프롬프트에서 자연스럽게 참조합니다. "런칭에 대해 Slack에서 팀이 뭐라 했는지 확인해줘"처럼요.

> 커넥터가 없는 도구(내부 대시보드, 벤더 포털, 로그인 뒤 웹 앱)는 **Claude in Chrome**이 다리 역할을 합니다 (Module 3에서 다룸).

## 권한 모델

폴더를 고르면 그 안의 읽기·쓰기가 자동 허가됩니다. 그 안의 작업에 대해 Cowork는 두 권한 모드를 가집니다.

- **Ask before acting (기본)** — 외부에 영향을 주는 각 행동(이메일 전송, 메시지 게시, 파일 공유) 전에 확인을 구합니다.
- **Act without asking** — 그런 행동에도 멈추지 않습니다. 신뢰하는 도구·작업에만 쓰세요.

두 모드 모두에서 **파일을 영구 삭제하기 전에는 항상 확인**하며, 이 확인은 건너뛸 수 없습니다. 또한 커넥터/MCP 접근 범위와 웹 접근(관리자가 웹 검색을 끄거나 방문 사이트 제한 가능)도 사용자가 통제합니다.

## 지금 해보기

1. 실제 작업이 든 폴더를 고른다 (원치 않으면 내용을 새 폴더로 복사).
2. 앱 하나를 연결한다 (보통 이메일·클라우드 저장소가 가장 가치 큼).
3. 폴더 내용을 요약해달라고 요청한다.
4. 결과를 내 지식과 대조해 본다.

## 핵심 정리

- Cowork는 Claude Desktop app 안에서 동작하며, 작업 폴더 지정이 가장 중요한 설정입니다.
- 커넥터로 이메일·메신저·저장소·CRM을 연결하면 실제 데이터로 작업할 수 있습니다.
- 기본 권한 모드는 "행동 전 확인"이며, 삭제는 항상 확인 — 그래서 큰 작업을 안심하고 위임할 수 있습니다.
