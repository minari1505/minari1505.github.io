## 학습 목표

- Claude Cowork를 정의하고, Claude와 일하는 방식으로서 무엇이 다른지 설명하기
- Cowork를 "내 환경(내 파일, 내 앱, 내 도구) 안에서 일하는 Claude"로 이해하기
- Chat, Cowork, Code를 구분하고 언제 무엇을 써야 할지 알기

## Cowork란 무엇인가

Cowork는 Claude Desktop app 안의 한 모드로, Claude가 나와 **나란히 작업을 수행**하는 방식입니다. 컴퓨터의 폴더를 지정하고, 작업이 있는 앱(Gmail, Slack, Google Drive, 캘린더)을 연결한 뒤, 원하는 결과를 설명합니다. 그러면 Claude가 계획을 세우고, 단계를 밟고, 필요한 도구와 파일을 쓰고, 실제 결과물을 다시 폴더에 저장합니다.

가장 중요한 멘탈 모델: **Cowork는 "채팅"이 아니라 "위임"입니다.** Chat은 여전히 강력합니다 — 문제를 함께 생각하고, 초안을 다듬고, 브레인스토밍하고, 질문하는 곳입니다. Cowork가 새로 더하는 것은, **하나의 작업 전체**(맥락 수집, 분석, 파일 생성, 도구 사용)를 통째로 맡기고 완성된 결과를 돌려받는 능력입니다.

## Chat과 더 깊은 차이

- **Chat은 턴 단위**입니다. 도구를 끌어올 수 있지만 단계들은 사용자가 직접 엮습니다. 짧은 대화와 빠른 사고에 최적화돼 있습니다.
- **Cowork는 하나의 작업 전체를 지속**합니다. 여러 도구를 동시에 끌어오고, 다단계 계획을 수행하며, 완성된 산출물을 돌려줍니다. 이것이 Chat에 파일을 업로드하는 것만으로는 메울 수 없는 간극입니다.

### 좋은 Cowork 작업의 형태

> 지난 분기 가격 정책에 대해 우리가 무엇을 결정했지? Q3 Sales 폴더의 회의록, 세일즈 팀 이메일, 리더십 Slack, 제안 덱을 모두 살펴보고, 임원진에 보낼 1페이지 요약으로 정리해줘.

로컬 파일 + 여러 도구 + 여러 단계 + 실제 결과물 — 이것이 Cowork가 대신 처리해주는 작업의 모양입니다.

## Cowork는 내 환경 "안에서" 일한다

- **내 파일에서** — 지정한 폴더의 내용을 읽고, 완성된 출력을 다시 씁니다.
- **내 앱에서** — 연결한 이메일·캘린더·메신저·드라이브·CRM에서 맥락을 끌어옵니다.
- **내 브라우저에서** — 커넥터가 없는 웹 도구는 Claude in Chrome이 페이지를 직접 읽고 조작합니다 (대시보드, 포털, 로그인 뒤의 무언가).
- **내 도구로** — 설명만 하는 게 아니라 직접 행동합니다.

## Chat vs. Cowork vs. Code

- **Chat** — Claude와 함께 "생각"하기. 질문, 브레인스토밍, 탐색적 사고, 한 번에 도구 하나.
- **Cowork** — Claude에게 "위임"하기. 여러 도구를 가로질러 작업을 지속하고 완성된 산출물을 만드는 워크스페이스.
- **Code** — Claude와 함께 "소프트웨어 만들기". 코드베이스 안에서 터미널·git 접근으로 동작하는 개발 도구.

대부분의 지식 노동자는 Chat과 Cowork에서 살며, 둘을 매우 다른 용도로 씁니다.

## 통제권은 사용자에게

Claude는 시작 전에 계획을 보여주고, 기본적으로 중요한 행동(전송, 삭제, 공유) 전에 확인을 구하며, 언제든 방향을 조정하게 합니다.

## 핵심 정리

- Cowork는 "답"이 아니라 "작업"을 위해 만들어진 Claude로, 로컬·클라우드·브라우저 어디서든 내가 일하는 곳에서 함께합니다.
- 핵심 전환은 "채팅"에서 "위임"으로 — 평소 남에게 물어보던 일이 아니라, 평소 내가 직접 하던 일을 맡기는 것입니다.
- 다단계·파일 기반·다중 도구 작업을 처음부터 끝까지 처리하도록 설계됐고, 통제권은 항상 사용자에게 있습니다.
