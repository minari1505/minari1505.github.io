## 학습 목표

- Cowork에 맞는 세 가지 작업 패턴(다단계, 파일 기반, 다중 도구) 알아보기
- `/schedule`로 예약 작업 설정하기
- Dispatch로 휴대폰에서 작업 보내기

## Cowork에 맞는 작업의 조건

모든 작업이 Cowork에 맞는 건 아닙니다. 대부분은 다음 세 패턴으로 설명됩니다.

### 패턴 1: 다단계 작업

여러 단계가 필요한 작업이면 좋은 후보입니다. 한 번의 프롬프트로 Claude가 맥락을 모으고, 소스를 비교하고, 추가 조사를 하고, 요약 초안을 쓰고, 결과물을 포맷합니다.

- 한 주치 고객 피드백 이메일을 테마별로 분류(각 테마 예시 인용 포함)
- 세 보고서와 스프레드시트에서 수치를 뽑아 하나의 인터랙티브 대시보드로 조립

### 패턴 2: 실제 파일에 사는 작업

목표가 실제 산출물(Word, 스프레드시트, 덱, PDF)을 원하는 형식으로 내보내는 것이면 잘 맞습니다. 입력도 실제 파일이라, Claude가 폴더의 기존 파일을 읽고 수정해 다시 저장합니다. Chat은 새 문서를 만들 수 있지만, Cowork는 **이미 있는 파일** 위에서 작업합니다.

- 표준 템플릿과 회의록에서 끌어온 포맷된 고객 제안서
- 원시 스프레드시트에서 조립한 월간 지표 보고서(차트 포함)

### 패턴 3: 여러 도구에 걸친 작업

이미 쓰는 시스템(Gmail, Slack, M365, 캘린더, CRM)에 걸친 작업입니다. Claude가 커넥터로 접근해 맥락을 모으고 행동합니다. Chat도 도구에 연결되지만, Cowork는 그것들을 **가로질러 계획하고 전체 시퀀스를 하나의 위임으로 실행**합니다.

- 캘린더 초대·참석자·회의록 기반으로 Outlook 후속 이메일 작성
- 런칭에 대한 Slack의 모든 언급을 검색해 업데이트로 종합

**진행 패널(Progress panel)**: Cowork는 작업을 단계로 쪼개 실행 중 각 단계를 보여줍니다. 지켜봐도 되고, 끝난 뒤 돌아와도 됩니다.

## 예약 작업 (Scheduled tasks)

작업을 한 번 설정해 일정에 맞춰 반복 실행합니다. 두 가지 방법:

- 처음부터: `/schedule`을 입력하고 작업과 주기를 설명하면 Claude가 프롬프트 초안을 작성 → 검토·수락.
- 더 자연스러운 방법: Cowork에서 작업을 한 번 하고 결과가 맞으면 `/schedule`로 그 과정을 반복 작업으로 전환. 주기(매시간/매일/평일/수동)를 정하고 프롬프트 검토 후 수락.

> 예약 작업은 컴퓨터가 켜져 있고 Claude가 실행 중일 때만 돕니다. 노트북이 닫혀 있거나 꺼져 있으면, 돌아왔을 때 Claude가 지연됐음을 알리고 작업을 집어 듭니다.

예: 금요일 오후 주간 리뷰, 매월 1일 지표 롤업, 아침 브리핑.

## Dispatch: 휴대폰에서 작업 보내기

Dispatch는 컴퓨터에서 떨어져 있을 때의 작업을 다룹니다. **Claude 모바일 앱에서 작업을 지정**하면, 작업은 데스크톱에서(같은 파일·커넥터·권한으로) 실행되고, 끝나면 푸시 알림을 받습니다. 데스크톱이 켜져 있고 로그인·Cowork 열림 상태일 때만 동작합니다.

> Dispatch는 research preview로 현재 Pro·Max plan에서 제공됩니다. Enterprise는 아직 없을 수 있고, 활성화된 곳에서는 IT 관리자가 설정합니다.

## 핵심 정리

- 다단계·파일 기반·다중 도구 — 이 세 패턴이 무엇을 맡길지 알려줍니다.
- 예약 작업과 Dispatch는 언제·어디서 위임이 일어날지를 정합니다.
- 판단은 여전히 사용자 몫입니다. Claude는 계획·실행·질문하고, 무엇을 보내고 공유할지는 사용자가 정합니다.
