## 학습 목표

- 코딩 assistant가 단순한 코드 생성기와 어떻게 다른지 이해하기
- Claude Code가 tool use를 통해 파일, 명령, 코드베이스와 상호작용하는 방식 파악하기
- Claude의 강한 tool use 능력이 Claude Code의 보안성, 확장성, 지속성을 어떻게 높이는지 정리하기

## 코딩 assistant는 무엇인가?

코딩 assistant는 단순히 코드를 써주는 모델이 아닙니다. 실제 개발자가 문제를 해결할 때처럼 **맥락을 모으고, 계획을 세우고, 행동을 실행하는 시스템**에 가깝습니다.

예를 들어 에러 메시지를 보고 버그를 고치는 작업을 맡기면, 좋은 코딩 assistant는 보통 다음 과정을 거칩니다.

1. **맥락 수집**: 에러가 무엇을 의미하는지, 어떤 파일과 코드가 관련 있는지 확인
2. **계획 수립**: 어떤 코드를 바꾸고 어떤 테스트로 검증할지 결정
3. **실행**: 파일을 수정하고 명령을 실행해 결과 확인

여기서 중요한 점은 1번과 3번이 모델 내부 텍스트 처리만으로는 불가능하다는 것입니다. 코딩 assistant는 파일을 읽고, 문서를 찾고, 명령을 실행하고, 코드를 편집할 수 있어야 합니다.

## 언어 모델만으로는 부족한 이유

언어 모델은 기본적으로 텍스트를 입력받아 텍스트를 출력합니다. 혼자서는 로컬 파일을 읽거나 셸 명령을 실행하거나 저장소를 탐색할 수 없습니다.

예를 들어 순수한 언어 모델에게 `"main.go 파일에 어떤 코드가 있어?"`라고 묻는다면, 모델은 실제 파일 시스템에 접근할 수 없으므로 내용을 알 수 없습니다. Claude Code 같은 코딩 assistant는 이 한계를 **tool use**로 해결합니다.

## Tool use가 동작하는 방식

Tool use는 모델이 외부 행동을 직접 수행하는 것이 아니라, "어떤 도구를 어떤 입력으로 호출하고 싶다"고 구조화된 요청을 만들게 하는 방식입니다.

간단한 흐름은 다음과 같습니다.

1. 사용자가 `"main.go 파일에 어떤 코드가 있어?"`라고 요청
2. Claude Code가 모델에게 사용할 수 있는 도구와 사용법을 함께 제공
3. 모델이 `ReadFile: main.go`처럼 파일 읽기 도구 호출을 요청
4. Claude Code가 실제로 파일을 읽고 결과를 모델에게 다시 전달
5. 모델이 파일 내용을 바탕으로 최종 답변 생성

즉, 모델은 여전히 텍스트를 생성하지만, Claude Code가 그 텍스트를 해석해 실제 파일 읽기, 파일 쓰기, 명령 실행 같은 작업을 수행합니다.

```text
사용자 요청
  ↓
Claude가 필요한 도구 선택
  ↓
Claude Code가 실제 도구 실행
  ↓
실행 결과를 Claude에게 전달
  ↓
Claude가 다음 판단 또는 최종 답변 생성
```

이 구조 덕분에 Claude Code는 코드베이스를 읽고, 수정하고, 테스트하며, 여러 단계의 개발 작업을 이어갈 수 있습니다.

## Claude의 tool use가 중요한 이유

모든 모델이 도구를 같은 수준으로 잘 쓰는 것은 아닙니다. Claude는 도구의 의미를 이해하고, 여러 도구를 조합해 복잡한 작업을 수행하는 능력이 강합니다.

이 능력은 Claude Code에서 세 가지 장점으로 이어집니다.

### 1. 더 어려운 작업을 처리할 수 있음

복잡한 개발 작업은 보통 하나의 도구 호출로 끝나지 않습니다.

예를 들어 버그 수정은 다음처럼 이어질 수 있습니다.

1. 에러 로그 읽기
2. 관련 파일 검색
3. 코드 읽기
4. 수정 계획 세우기
5. 파일 편집
6. 테스트 실행
7. 실패하면 다시 원인 분석

Claude가 도구를 잘 쓸수록 이런 multi-step 작업을 더 자연스럽게 수행할 수 있습니다. 이미 본 적 없는 새 도구가 추가되더라도, 도구 설명을 읽고 목적에 맞게 사용할 가능성이 높아집니다.

### 2. 확장 가능한 플랫폼이 됨

Claude Code의 도구는 고정된 기능 목록으로 끝나지 않습니다. 새 도구를 추가하면 Claude가 그 도구를 workflow에 포함할 수 있습니다.

예를 들어 문서 변환, 배포, 이슈 조회, 브라우저 자동화, 내부 API 호출 같은 도구를 붙이면 Claude Code는 기존 코딩 작업을 넘어 팀의 실제 개발 환경과 연결됩니다.

강한 tool use는 Claude Code를 단순한 "코드 작성 도구"가 아니라, 계속 확장 가능한 개발 플랫폼으로 만듭니다.

### 3. 더 나은 보안 구조를 만들 수 있음

일부 코딩 assistant는 코드베이스 전체를 미리 인덱싱하거나 외부 서버에 대량 전송해야 제대로 동작합니다. Claude Code는 필요한 순간에 필요한 파일을 도구로 읽는 방식이므로, 전체 코드베이스를 항상 외부로 보내는 구조를 피할 수 있습니다.

물론 Claude Code도 사용 중 선택된 파일 내용이나 명령 결과를 모델에 전달할 수 있습니다. 하지만 "필요한 맥락을 도구로 찾아 읽는다"는 구조는 불필요하게 모든 코드를 한꺼번에 보내는 방식보다 더 통제하기 쉽습니다.

## 내장 도구의 역할

Claude Code에는 일반적인 개발 작업에 필요한 기본 도구들이 포함되어 있습니다. 대표적으로 다음과 같은 작업을 할 수 있습니다.

- 파일 읽기
- 파일 생성과 수정
- 디렉터리 탐색
- 코드 검색
- 셸 명령 실행
- 테스트 실행

중요한 점은 도구 하나하나보다, Claude가 이 도구들을 **어떤 순서로 조합하는가**입니다. 좋은 코딩 assistant는 "파일을 읽어라", "테스트를 실행해라" 같은 단일 명령만 수행하는 것이 아니라, 목표를 달성하기 위해 필요한 도구 사용 흐름을 스스로 구성합니다.

## 정리

- 코딩 assistant는 언어 모델 + 도구 실행 시스템으로 이루어집니다.
- 개발 작업은 맥락 수집, 계획 수립, 실행으로 진행되며, 맥락 수집과 실행에는 외부 도구가 필요합니다.
- Tool use는 모델이 파일 읽기, 명령 실행, 코드 수정 같은 외부 행동을 요청할 수 있게 하는 구조입니다.
- Claude는 도구 사용 능력이 강해 복잡한 multi-step 코딩 작업과 새 도구 확장에 유리합니다.
- Claude Code는 필요한 맥락을 도구로 읽는 방식이라, 전체 코드베이스를 무조건 인덱싱하는 구조보다 더 통제 가능한 보안 모델을 만들 수 있습니다.
